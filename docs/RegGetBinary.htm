%RegGetBinary

Description

 Get a binary (REG_BINARY) value from the Windows Registry

Interface

 
status = %RegGetBinary(reghdl ,hive ,key ,value ,data ,size)
    or
xcall RegGetBinary(reghdl ,hive ,key ,value ,data ,size)

Arguments

 reghdl A registry session handle returned by RegStartSession. (i4)
hive The Registry hive in which the key that you wish to create resides.  This can be one of the following (as defined in PSG:registry.def:
HKEY_CLASSES_ROOT
HKEY_CURRENT_CONFIG
HKEY_CURRENT_USER
HKEY_LOCAL_MACHINE
HKEY_USERS
HKEY_PERFORMANCE_DATA (Windows NT only)
HKEY_DYN_DATA         (Windows 95 and Windows 98 only)
key The registry key (directory) from which the value will be retrieved. (a)
value The name of the value to be retrieved from the specified key in the specified hive. (a)
data Returned with the handle of a dynamic memory section which has been created and which contains the binary data from the specified value in the Registry. Your program should de-allocate this memory when it has finished with the returned data. (i4)
size Returned with the number of bytes of data contained in the dynamic memory section whose handle was returned in the data argument. (n)

Return Value

 If successful the routine returns REG_SUCCESS (0).  Otherwise one of the REG_ERR_ error states is returned (as defined in PSG:registry.def).

Remarks

 This routine retrieves a REG_BINARY value from the specified key in the Registry. If the specified key or value is not found then a failure status is returned, otherwise the handle of a dynamic memory section containing the binary data from the registry is returned, along with the length of that data in bytes.  It is the responsibility of the programmer to deallocate the dynamic memory section when the data retrieved is finished with (using %MEM_PROC(DM_FREE,handle).

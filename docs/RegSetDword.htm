%RegSetDword

Description

 Set an integer (REG_DWORD) value in the Windows Registry

Interface

 
status = %RegSetDword(reghdl ,hive ,key ,value ,data [,flush])
    or
xcall RegSetDword(reghdl ,hive ,key ,value ,data [,flush])

Arguments

 reghdl A registry session handle returned by RegStartSession. (i4)
hive The Registry hive in which the key that you wish to create resides.  This can be one of the following (as defined in PSG:registry.def:
HKEY_CLASSES_ROOT
HKEY_CURRENT_CONFIG
HKEY_CURRENT_USER
HKEY_LOCAL_MACHINE
HKEY_USERS
HKEY_PERFORMANCE_DATA (Windows NT only)
HKEY_DYN_DATA         (Windows 95 and Windows 98 only)
key The registry key (directory) in which the value will be created or updated. (a)
value The name of the value to be created or updated in the specified key in the specified hive. (a)
data A numeric variable or literal containing the data to be stored in the named registry value. (n)
flush An optional flag specifying that the new or updated Registry value should be flushed to disk immediately.  By default, the registry is flushed to disk only periodically (as determined by the operating system) and at system shutdown.  To cause the new or updated value to be flushed to disk immediately, pass REG_FLUSH. (n)

Return Value

 If successful the routine returns REG_SUCCESS (0).  Otherwise one of the REG_ERR_ error states is returned (as defined in PSG:registry.def).

Remarks

 This routine sets a REG_DWORD (4-byte integer, i4) value in the specified key in the Registry. If the value is not found, it will be created. If the key (or parent keys) are not found, they will be created. In other words, this routine can create a new value, several levels deep in a totally new heirarchy in the Registry.
